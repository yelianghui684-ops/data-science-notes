# Manage as a Machine

## 一句话概括
把组织(和自己)看成一台产出结果的机器:站在机器上方俯视它、诊断它、改进它。

## 核心要点
- 机器 = **文化 + 人**;产出不符合目标时,修机器而不是就事论事救火
- 循环:感知问题 → 诊断根因 → 设计改进 → 推动执行(五步流程的组织版 → [[The Five-Step Process]])
- 关键姿态:你既是机器里的工人,也是机器的设计师 —— 要能跳出来看自己
- 工具化:把原则变成工具与协议(棒球卡、点收集器、痛苦按钮),让文化可执行
- 同构:MLOps 的"监控 → 漂移诊断 → 迭代"就是把模型当机器管(→ [[MLOps]])
